Extension points for satellite dtype crates (zax-time, zax-geo, …).

The dtype model is Primitive + Extension (checkpoints/dtype-external.md): core owns the primitive dispatch key and the capability trait vocabulary; everything with meaning (datetime, categorical, string, geometry) is an ExtensionDType living in a satellite crate, paired with an ArrayData payload (ArrayPayload::Extension) and an ExtensionScalar host value.

Core reasons about an extension dtype by capability, not identity (“ask, don’t match”): an op is rejected at plan time iff the capability is absent. BinaryCapability is the only capability for now; Temporal / Order land additively.

ScalarValue
Host-side typed scalar across both dtype arms — Value is the primitive-only tagged union; this adds the extension arm. The dtype is derived from the value (mirroring ArrayPayload::dtype), not stored.

Build this <op> other when this dtype owns the op. The direct analogue of Python’s __add__/__radd__ operand-coercion protocol; apply runs with no brick — it infers the output dtype, inserts operand casts, and returns a built ArrayExpr.

ExtensionScalar
A satellite-owned rank-0 value. Surfaces at the host boundary — constructing a literal, or materializing a 0-d result (.item()) — and as a binary operand (Operand::Scalar): the owning capability resolves it at plan time, to its storage primitive or to an extension-dtype literal node (see BinaryCapability).
