Expand description

An externally-owned logical dtype.

Carries no compute methods: a dtype is identity plus plan-time opt-in capability gates (e.g. “i support binary ops”). Carries allocation primitives, since a DType is the only concrete handle we have at runtime. ArrayPayload::empty() cannot be implemented because Rust.

fn binary(&self) -> Option<&dyn BinaryCapability>

The binary-op capability, or None if this dtype advertises no binary ops (then every binary() involving it is a plan-time error). FIXME: we may need to be smarter here, but for now… this is OK.

fn empty(&self, shape: &[usize]) -> Box<dyn ArrayData>

Allocate a writable host array of shape with unspecified contents — dtype-driven construction (np.empty(shape, dtype)). Lives on the dtype because no array exists yet to dispatch through; sinks populate the result via ArrayData::write_range.

Source

fn full( &self, fill: &dyn ExtensionScalar, shape: &[usize], ) -> Box<dyn ArrayData>

Allocate a writable host array of shape filled with fill — np.full(shape, fill, dtype). Infallible like Self::empty: fill belonging to this dtype is a caller-validated precondition, so the satellite downcast is a genuine invariant (expect). Serves sink prefill (extension fills) and rank-0 literal materialization.
