Expand description

Trait for user-facing array variants (Array, BlockSparseArray). Required method: Self::expr. Everything else has a default that reads through the expr.

fn to_sink(&self, sink: Arc<dyn ArraySink>) -> ArrayResult<Array>

Wrap this array in a terminal ArraySink, yielding an Array ready for lower/plan/execute. Fallible: the sink validates the output dtype (e.g. a fill value of the wrong dtype) at construction.

Source

fn load( &self, executor: impl Executor, memory_budget: usize, ) -> ArrayResult<Array>
where Self: Sized,

Materialize this (lazy) array and re-wrap the result as a fresh in-memory Array. Runs the full pipeline — optimize → sink → lower → plan → execute — into an InMemorySink, then feeds the assembled payload back through Array::from_payload. memory_budget bounds peak memory during execution (see crate::lower::lower).

Dtype-agnostic: dense and extension payloads round-trip identically.

Source

fn payload(&self) -> Option<&ArrayPayload>

The backing in-memory payload, when this array is a bare transparent InMemorySource — i.e. one produced by Array::from_payload or materialized by Self::load. None for any other expression (constant fills, opaque/IO sources, or arrays carrying unevaluated ops).
