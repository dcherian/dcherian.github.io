Expand description

Transparent source — generates any requested brick on demand, cheaply and infallibly. The trait behind functional arrays: an impl is a generation rule (constant, range, function) plus its shape and dtype.

read is infallible by contract — validation happens at expression-time. For now, fallibe sources should be OpaqueSource. So out-of-range (for example) is detected at plan-time For the moment, there is no cost hook in favor of numel × itemsize from grid + dtype.
