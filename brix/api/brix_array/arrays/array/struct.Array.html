pub struct Array {
    expr: Arc<ArrayExpr>,
}
Expand description

User-facing logical array — a thin handle over Arc<ArrayExpr>. All shape/dtype/dims/layout info lives on the underlying ArrayExpr. Array exists to give the user a clean and familiar API surface Methods construct specializations of underlying ArrayExpr.

pub fn from_ndarray<T: Element>(data: ArrayD<T>, axes: &[&str]) -> Self

Wrap an in-memory ndarray of any primitive dtype as a transparent dense source.

use brix_array::{Array, DType};
use ndarray::array;

let x = Array::from_ndarray(array![[1.0, 2.0], [3.0, 4.0]].into_dyn(), &["i", "j"]);
assert_eq!(x.expr().dtype, DType::F64);

let n = Array::from_ndarray(array![1i64, 2, 3].into_dyn(), &["i"]);
assert_eq!(n.expr().dtype, DType::I64);

pub fn isel<K: Into<AxisName>>( self, indexers: impl IntoIterator<Item = (K, Indexer)>, ) -> ArrayResult<Self>

Positional selection — accepts a mixed map of slice and orthogonal integer indexers per axis. Decomposes into a chain at the Array layer: emits ArrayOp::Slice for slicer axes, then ArrayOp::Oindex for gather axes. Distinct axes commute, so the order is fixed (slice first — cheaper, narrows the input for the subsequent gather). All axis/bounds validation happens in the ArrayExpr::{select, oindex} constructors.

where Self: Sized,

Materialize this (lazy) array and re-wrap the result as a fresh in-memory Array. Runs the full pipeline — optimize → sink → lower → plan → execute — into an InMemorySink, then feeds the assembled payload back through Array::from_payload. memory_budget bounds peak memory during execution (see crate::lower::lower).
