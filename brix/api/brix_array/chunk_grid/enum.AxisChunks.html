Expand description

Per-axis chunk size encoding for a rectilinear chunk grid. Three variants covering progressively more general cases:

  • AxisChunks::Uniform — every chunk identical size. O(1) lookups. Produced by ChunkGrid::new when shape divides the chunk target.
  • AxisChunks::RLE — run-length encoded (size, count) pairs. Produced when slicing or boundary-tail patterns leave a few segments.
  • AxisChunks::Variable — fully explicit Vec<usize>.

pub fn coarsen(&self, divisor: usize) -> ChunkGridResult<Self>

Coarsen by divisor (the element-space window Frame::Window {s,s}). Two regimes, by how the window relates to the chunk size:

  • within-brick (divisor divides the chunk): the window fits inside a chunk, so each chunk of size c becomes c/divisor and the chunk count is unchanged. The fold is intra-brick (k=1).
  • cross-brick (divisor is a multiple of the chunk size, Uniform only): the window spans k = divisor/size whole bricks and collapses to one element, so k bricks fold into one size-1 output brick → ceil(n_chunks/k) chunks.

Ceil semantics apply only to this cross-brick fold. Element-level misalignment within a chunk (within-brick size % divisor != 0, or an incommensurate chunk size) still hard-errors — a window can’t span a partial chunk.
