The uniform contract over payload data — what every payload kind supports regardless of storage representation. TODO: break this up in to more super-traits when it makes sense. this will let us be more explicit about the contract for each use-case e.g. ArrayData is too much for SinkBuffer; Note: traits cant have constructors, so empty is not here but on the DType. Write data into the element-space region ranges (one per axis) — the sink population path, only ever called on a buffer the sink exclusively owns (core-allocated, or via crate::extension::ExtensionDType::empty). Accept the payloads this data kind can ingest; reject the rest.
