Kernels adapt from Brick to Array.

Every mid-IR Blockwise node carries an Arc<dyn Kernel>. The four kernels here are the lowering targets for four array-level ops:

  • Binary → BinaryKernel
  • Unary → UnaryKernel,
  • Slice → SliceKernel,
  • Oindex → GatherKernel.

Indexing kernels — the executor-side targets for ArrayOp::Slice (positional slicing) and ArrayOp::Oindex (orthogonal integer indexing).
ops
Elementwise kernels — thin wrappers around BinaryOp::apply and UnaryOp::apply. Output shape at runtime comes from input bricks’ payloads, but the kernel still carries its output ChunkGrid so its cost closure can derive brick shape from coord.
