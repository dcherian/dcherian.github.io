The array-layer brick payload and its uniform data contract.

ArrayPayload is the closed kind enum the runtime moves around; ArrayData is the structural contract trait every kind implements — DenseArray (dense.rs) and

  • externally provided extension arrays (ArrayPayload::Extension).

Enums§

ArrayPayload
Array-layer brick payload. The arms are the closed kind tag; the data behind each arm answers the uniform ArrayData contract.
ArrayPayloadKind
Plan-time tag for what kind of bricks a source/operand will emit. The runtime dual of ArrayPayload — variants must stay in lock-step. Unused at the moment. I expect we may want this kind of thing at plan-time for cross-array ops e.g. tensordot(sparse, dense).
BroadcastMode
How an input axis broadcasts up to the target extent in [ArrayPayload::align].
PrimitivePayload
Primitive-only projection of ArrayPayload — what the monomorphized kernels (ops.rs, the cast kernel) accept. Obtained via ArrayPayload::as_primitive, which rejects Extension once at the kernel boundary instead of every body carrying a dead match arm. (Single-armed today; we expect a primitive-dtype Sparse arm might join it.)
WriteFrame
Mutable, framed view of an accumulator’s state buffer. An enum over the kind axis (so a future sparse-state accumulator adds a Sparse arm), but generic over the element type T rather than dtype-tagged: the writer is an accumulator that already dispatched on its own dtype, so T is statically known here.
