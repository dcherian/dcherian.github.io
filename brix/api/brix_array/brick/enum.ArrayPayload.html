Enum ArrayPayload

pub enum ArrayPayload {
    Dense(DenseArray),
    Extension(Arc<dyn ArrayData>),
}
Expand description

Array-layer brick payload. The arms are the closed kind tag; the data behind each arm answers the uniform ArrayData contract.

pub fn get_frame_mut<T: Element>( &mut self, axes_idx: &HashMap<usize, usize>, ) -> WriteFrame<'_, T>

Grab a mutable frame at axes_idx, typed as T.

axes_idx maps each framed axis (by positional index into this payload) to the frame index along that axis. Non-framed axes (loop dims) are kept at full extent.

Mutable access through ArcArray is copy-on-write; callers (accumulator state) hold the sole reference to their state buffer by construction — they allocate it and never hand out clones while folding — so the copy never fires.

Panics if T does not match the payload’s dtype — callers (accumulators) dispatch on the dtype first, so the tag always matches.

pub fn axis_chunks_iter( &self, axis: usize, size: usize, ) -> impl Iterator<Item = ArrayPayload> + '_

Iterate sub-views by splitting axis into contiguous chunks of size size. Matches ndarray’s axis_chunks_iter semantics — the last chunk may be smaller.

Source

pub fn as_frame_iter( &self, view: HashMap<usize, Frame>, ) -> impl Iterator<Item = (HashMap<usize, usize>, ArrayPayload)> + '_

Iterate (axes_idx, slot) pairs cut out of this brick.

view carries one entry per reduce axis, keyed on the brick-position of that axis. The iterator yields the cartesian product of per-axis frames in lexicographic order of axis position.

pub fn kind(&self) -> ArrayPayloadKind

Plan-time tag this payload corresponds to. Must stay in lock-step with ArrayPayloadKind. Used when handing runtime data back to plan-time code (optimizer, cost models) that key off ArrayPayloadKind.

Source

pub fn from_payload(payload: &dyn BrickPayload) -> &ArrayPayload

Downcast a dyn BrickPayload to &ArrayPayload. Array-layer kernels are the sole producers of array-side payloads, so this is always Some in array-side code paths — expect is appropriate.

Source

pub fn as_primitive(&self) -> BrixResult<PrimitivePayload<'_>>

Project to the primitive arms, or error for Extension — the single gate at the boundary of every monomorphized primitive kernel. Bodies downstream match PrimitivePayload exhaustively, with no dead arms.

Source

pub fn dense_view<T: Element>(&self) -> Option<ArrayViewD<'_, T>>

Borrow a Dense payload’s buffer as a concrete ArrayViewD<T>. None for Extension payloads or a dtype mismatch. Callers in ops / accumulators dispatch on dtype first, so the tag always matches.

Source

pub fn to_dense_dyn(&self) -> BrixResult<DenseArray>

Materialize to a dtype-tagged DenseArray. Dense is a zero-copy buffer share. Extension is an error — a satellite payload has no primitive dense form; consumers reach it by downcast.
