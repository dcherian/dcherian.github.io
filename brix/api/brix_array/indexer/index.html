Array-layer positional selection — the user-facing isel vocabulary.

Indexer describes how a single axis is to be subset. Three variants covering the three array-level selection styles:

  • Slicer(Slice) — contiguous-with-stride slice. Preserves axis identity. Lowered to ArrayOp::Slice then SliceKernel, which carries a positional Vec<Option<Slice>> into the mid-IR.
  • Outer(Vec<usize>) — orthogonal integer indexing; output gathers the named positions in order. Lowered to ArrayOp::Oindex then GatherKernel. Indices must be sorted by source chunk (planner-checked).
  • Vectorized — pointwise integer indexing across axes (vindex). Not yet implemented; future first-class op per vindex.md.

Mixed-variant .isel(...) calls decompose at the array layer into a chain of ArrayOp::Slice (slicer axes) then ArrayOp::Oindex (outer axes); the mid-IR sees Slice directly, never the enum.
