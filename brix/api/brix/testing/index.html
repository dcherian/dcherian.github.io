Test utilities. Stable for in-tree integration tests and benchmarks; downstream crates that depend on brix for their own pipeline tests can use these too.

Currently a single helper: RecordingSource, an OpaqueSource that records every get_brick coord it’s asked for. The strongest way to test slice pushdown — assert which source chunks the executor fetched, not just the final values.

Structs§

BlockSparseSource
Block-sparse OpaqueSource. Emulates a Zarr-flavored array where some chunks are stored as dense blocks and the rest are absent (resolving to fill_value at read time).
RecordingSource
OpaqueSource backed by an in-memory ArrayD<f64> that records every get_brick query’s chunk-coord (derived from the requested bounds against the source’s grid).
