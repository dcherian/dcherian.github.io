pub fn assert_planner_invariants(plan: &PhysicalPlan)
Expand description

Validate structural invariants the planner must uphold on a PhysicalPlan.

Panics on violation — these are planner bugs, not user errors. Called under #[cfg(debug_assertions)] at the tail of planner::plan so the whole plan is validated once at build time; the executor then trusts it and short-circuits with .expect("invariant: ...") on the same conditions as defence-in-depth for release builds.

Checks covered:

  1. Every Loop/Init/Apply/Finalize/WriteSink has a Domain ancestor.
  2. Every Loop { axis } has axis in the nearest enclosing Domain’s iter_space.names.
  3. Every Init/Finalize { node } references a stateful compute node
  4. Every Apply { node, .. }: nodes[node] is not NodeInfo::Sink.
  5. Every WriteSink { node, sink }: nodes[sink] is NodeInfo::Sink, nodes[node] exists.
  6. Every InitSink/FinalizeSink { sink }: nodes[sink] is NodeInfo::Sink.
  7. Every NodeName referenced exists in plan.nodes.
  8. plan.nodes contains at least one NodeInfo::Sink.

Temporal invariants (Init precedes Apply/Finalize for a stateful node; WriteSink preceded by a workspace-producing step) are deferred — they require a traversal-order model worth a separate pass.
