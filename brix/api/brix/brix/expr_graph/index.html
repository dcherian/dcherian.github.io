Graph representation of a logical expression.

The expression layer is a DAG (shared subexpressions via Arc<Expr>), but Expr encodes that DAG as a recursive tree of Arc pointers. For planning we need O(1) consumer queries (“who reads this reduce’s output?”) and a stable node identity that survives rewrites. StableDiGraph gives both.

Edges point producer → consumer. Toposort yields producers first, matching the post-order the planner currently relies on.

The node/edge split mirrors the field split on Expr / Input:

  • ExprNode carries the intrinsic info about what the node produces (dims, op, label).
  • ExprEdge carries the per-edge consumption metadata (access, port).
