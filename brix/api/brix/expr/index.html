Mid-level IR — core lower-level vocabulary for expressing ops on a chunk grid.

Expr is an expression describing an operation on a Brick grid. It carries axis names (dims) and chunk counts (grid) only — element extents, dtype, and brick kind all live on ArrayExpr (the array-level IR). The lowering pass [brix_array::lower::lower] is the boundary between the two and bakes the element-sized ChunkGrid into each concrete Source / Kernel / Accumulator impl.

Frame
How a consumer frames a producer’s bricks along one axis — the brick-space analog of the array-level [brix_array::expr::Frame], in chunk coords. This is a structural iteration-space transform: the planner expands it into an operation applied to the input crate::layout::NamedLayout. What the Op does with the resulting tile (sum it in a Reduce, map over it in a Blockwise) is the Op’s concern, not the frame’s. Never a point transform (the affine view + AxisOverride cover those); identity (pass-through / broadcast) is the absence of an entry.

Type Aliases§

TranslatedAccessStack
An ordered per-axis override stack. The planner accumulates one per emission while descending consumer edges (downstream-most edge first), and AccessRelation::resolve_coord applies the entries in that order — so stacked overrides on one axis compose (+1 then +2 reads the producer at +3). Order is load-bearing; an axis may appear more than once.
