brix::array::chunk_grid

Module vindex 

Source
Expand description

Chunk-grid algebra for vectorized indexing (ArrayOp::Vindex → BlockwiseOp::VIndex).

Plan-time math: given an input ChunkGrid and a set of per-axis 1-D integer index arrays zipped into a single produced axis k, partition k into runs by source chunk (multi-axis chunk-coord tuple) and build the output ChunkGrid.

Contrast with super::oindex: oindex runs are independent per axis (the cartesian product reconstructs the output). Vindex zips the consumed axes — each k position picks one cell across all consumed axes simultaneously, so a “run” is a contiguous group of ks that map to the same multi-axis input chunk.

The narrow slice (see checkpoints/vindex.md):

  • indices are literals, known at plan time,
  • pre-sorted so that the per-run input-chunk-coord tuple is non-decreasing in lex order over the consumed axes,
  • hard error otherwise (no auto-sort here; see shuffle.md).
