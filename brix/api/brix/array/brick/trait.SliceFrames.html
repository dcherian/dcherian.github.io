Expand description

The single per-backend frame-extraction primitive.

Given per-axis ranges (axes not listed stay at full extent), produce a borrowed ReadFrame. Everything else on the frame-generation surface — ArrayPayload::view, ArrayPayload::axis_chunks_iter, ArrayPayload::as_frame_iter — is range arithmetic layered over this one method, so a new storage kind (sparse, extension) becomes reachable by adding one impl SliceFrames, not by touching the chunking or cartesian-product drivers.

This is also where the “view may not exist for this layout” escape hatch lives: a sparse backend that can’t form a cheap view for the requested ranges may materialize and return an owned Dense/Fill frame, eating the cost (see the ReadFrame note). That decision stays local to the backend.
