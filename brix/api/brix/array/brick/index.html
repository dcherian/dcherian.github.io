brix::array

Module brick 

Source
Expand description

Concrete brick payloads. DenseArray carries a materialized TypedArray (ArrayD<f64> / ArrayD<i64>); FillArray carries a constant Value plus its shape. Both implement BrickPayload so they can flow through the mid-IR uniformly.

Storage is dtype-generic via TypedArray (the dense kind’s typed buffer) and Value (a typed scalar). Data movement (slice / select / align / clone) is dtype-agnostic and dispatched through with_typed_array!; arithmetic lives in ops.rs / accumulators.rs and dispatches via dispatch_dtype!.

Borrowed, dtype-tagged view — the Dense arm of ReadFrame. Mirrors TypedArray for views. Consumers that know their T recover a concrete ArrayViewD<T> via TypedView::as_typed.
WriteFrame
Mutable, framed view of an accumulator’s state buffer — the write-side counterpart of ReadFrame. An enum over the kind axis (so a future sparse-state accumulator adds a Sparse arm), but generic over the element type T rather than dtype-tagged: the writer is an accumulator that already dispatched on its own dtype, so T is statically known here (no runtime tag to re-match — that’s the asymmetry with ReadFrame, whose producer doesn’t know the consumer’s T).
