Data types.

Lives in array/ because dtype is a property of the array data layer, not the structural mid-IR. Concrete array/ impls bake their dtype in at construction time.

Macros§

dispatch_dtype 🔒
Map a runtime DType to a compile-time type alias $t, then run $body once per arm. The trick: type $t = f64; inside the arm is a local alias the body uses as a turbofish parameter — no body rewriting, no T plumbing.
with_typed_array 🔒
Destructure a TypedArray, binding the inner ArrayD<T> to $arr and a local type alias $t to its element type, then run $body per arm.

Traits§

Binary
Capability umbrella for the BinaryOp family (add / sub / mul) plus the Zero/One identities reductions need. Unsigned-compatible — a future u8/u16 code-width dtype implements this, so binary ops / sum / cast stay open to it. The universal arithmetic bound; mirrors BinaryOp.
Element
Native primitive backing a TypedArray / Value arm. The static mirror of the runtime DType tag: T::DTYPE is the tag, from_array/from_scalar re-tag a concrete buffer/scalar, and from_value peels the tag back off.
Numeric
Fully numeric: both op families. The convenience bundle for code that needs everything; bodies that need only one family should bound on Binary or Unary directly so they don’t over-constrain (e.g. exclude unsigned).
Unary
Capability for the UnaryOp family (negation / absolute value). Signed only — Signed supplies both Neg and abs, so unsigned dtypes are correctly excluded. Mirrors UnaryOp.
