Array data layer — logical arrays, their mid-IR lowering, and the concrete brick payloads / kernels / accumulators they lower to.

The user-facing array kinds (Array, BlockSparseArray) and their shared ArrayLike contract live in arrays; this module root wires the submodules together and re-exports the public surface. The per-dtype trait instantiations live in generated/ (a folder convention, not codegen).

Per-axis positional slicing. This kernel must record the input & output chunk grids: input_grid and grid respectively. WriteFrame
Mutable, framed view of an accumulator’s state buffer — the write-side counterpart of ReadFrame. An enum over the kind axis (so a future sparse-state accumulator adds a Sparse arm), but generic over the element type T rather than dtype-tagged: the writer is an accumulator that already dispatched on its own dtype, so T is statically known here (no runtime tag to re-match — that’s the asymmetry with ReadFrame, whose producer doesn’t know the consumer’s T).

Capability umbrella for the BinaryOp family (add / sub / mul) plus the Zero/One identities reductions need. Unsigned-compatible — a future u8/u16 code-width dtype implements this, so binary ops / sum / cast stay open to it. The universal arithmetic bound; mirrors BinaryOp.

Native primitive backing a TypedArray / Value arm. The static mirror of the runtime DType tag: T::DTYPE is the tag, from_array/from_scalar re-tag a concrete buffer/scalar, and from_value peels the tag back off.
