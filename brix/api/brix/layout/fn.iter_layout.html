Build a unit-stride iteration-space layout — every axis has stride 1.

compose works as (producer ∘ view)(iter) = producer(view(iter)). For each axis, view.eval(c) = c · view_stride. If we want the iter coord c to land directly at producer chunk c · producer_stride on that axis (with the producer’s stride providing the chunk-grid’s linear addressing), the view’s stride must be 1 — anything else scales c before it reaches the producer.

Use this for loop iter-space layouts: the planner’s outer_iter, Schedule::Domain.iter_space, and the layouts that compose consumes on the view side. For producer-side layouts — each node’s chunk grid, where the strides encode row-major chunk addressing so unravel_index can decode the composed result — use native_layout instead.
