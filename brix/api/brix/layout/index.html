CuTe-inspired hierarchical layout representation (Cecka 2026, arXiv:2603.02298).

Mirrors CuTe’s Shape:Stride representation:

  • IntTuple = HTuple(Z) from CuTe Definition 2.2
  • Layout = Shape:Stride from CuTe Definition 2.17

A layout maps coordinates to offsets into a linear ordering of elements via inner_product(coord, stride). Shapes and strides are congruent hierarchical integer tuples.

The code here was generated by pointing Claude at the paper and the online docs: https://docs.nvidia.com/cutlass/latest/media/docs/cpp/cute/index.html Unit tests map to worked examples in the paper. Function names map to existing CuTe function names. Two major changes:

  • Brix introduces NamedLayout attaching axis names to Layout.
  • Brix uses row-major (rightmost varies fastest) for index decomposition. CuTe uses column-major (leftmost varies fastest).

Build the natural coordinate (an IntTuple congruent to mode) for one named top-level mode, drawing each leaf from indices[(name, position)] in pre-order. A flat (Int) mode takes position 0; a tiled mode takes one position per leaf (CuTe get<0>, get<1>, …). A named coordinate, scattered per mode: (axis, mode_index) → component.
