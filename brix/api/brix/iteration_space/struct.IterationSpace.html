Struct IterationSpace

pub struct IterationSpace {
    pub layout: NamedLayout,
    pub occupancy: Option<Occupancy>,
    pub limits: HashMap<AxisName, usize>,
}
Expand description

Brick-grid iteration space:

The layout is a plain row-major (native) NamedLayout over chunk counts. This object carries no slicing or transposition (those live on edges, in AccessRelation).

A NamedLayout is not a full enough description of the iteration space, so we add two more fields.

An optional Occupancy selecting which coords are present. occupancy: None is dense; Some(_) is block-sparse. Occupancy stays outside the layout algebra — it is a filter applied by the executor (expressed using Schedule::Zip).

limits records per-axis predication bounds for logical_divided fold axes. For example, for coarsen/block-windows we use logical_divide(axis, w) which rounds the tile count up, and the last block may be smaller than the others. We need to record that information somewhere (not representable in layout). Instead of CuTe predication, we have a simpler limit imposition applied when grabbing the extent.

pub fn extent( &self, axis: &AxisName, mode_index: usize, indices: &NamedCoord, ) -> usize

Loop bound for axis’s mode_index-th sub-mode — the iteration count Schedule::Loop { axis, mode_index, .. } should run.

When this axis carries a predication limit in Self::limits (r.g. when logical_divide results in a smaller last brick), the bound is clipped to min(raw_extent, limit - outer·raw_extent). outer is the enclosing coarser mode’s current index read off indices.
