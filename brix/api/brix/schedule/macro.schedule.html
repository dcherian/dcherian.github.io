macro_rules! schedule {
    (@item Apply($tag:ident)) => { ... };
    (@item Apply($tag:ident, [$($inp:ident),* $(,)?])) => { ... };
    (@item Init($tag:ident)) => { ... };
    (@item Finalize($tag:ident)) => { ... };
    (@item InitSink($tag:ident)) => { ... };
    (@item WriteSink($tag:ident)) => { ... };
    (@item FinalizeSink($tag:ident)) => { ... };
    (@item Loop($axis:literal, $mode:literal, [$($body:tt),* $(,)?] $(,)?)) => { ... };
    (@item Loop($axis:literal, [$($body:tt),* $(,)?] $(,)?)) => { ... };
    (@item Domain([$(($ax:literal, $lo:literal..$hi:literal)),* $(,)?], $body:tt $(,)?)) => { ... };
    (@item Sequence([$($body:tt),* $(,)?] $(,)?)) => { ... };
    (@unwrap { $($inner:tt)+ }) => { ... };
    ($($tokens:tt)+) => { ... };
}

Build a Sketch from concise syntax. Body elements wrapped in {}.

ⓘ
schedule!(Loop("i", 0..2, [
    {Apply(DenseSource)},
    {Apply(Blockwise, [DenseSource])},
    {WriteSink(Blockwise)},
]))

Axis names are string literals (user data). Node tags like DenseSource remain bare idents (they correspond to NodeInfo variants, not user data).

Exported with #[macro_export] (not path-based pub use): the brix module is pub(crate), so a path export can’t cross the crate boundary, and the integration tests in tests/ build the expected Sketch with this.
