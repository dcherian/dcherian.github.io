The Brix Book

Extensions

This page describes the mechanism by which a satellite crate (zax-time, zax-geo, …) plugs a new logical dtype. Three contracts need to be implemented — a dtype, a scalar, and an array payload.

An extension dtype is identity plus plan-time capability gates — no compute methods. Runtime compute dispatches through the paired payload, never the dtype.

/// An externally-owned logical dtype.
///
/// Carries **no compute methods**: a dtype is identity plus plan-time
/// opt-in capability gates (e.g. "i support binary ops").
/// Carries allocation primitives, since a DType is the only concrete
/// handle we have at runtime. ArrayPayload::empty() cannot be implemented
/// because Rust.
pub trait ExtensionDType: DynEqHash + Send + Sync + Debug {
    /// Identity / display name (e.g. `"datetime64[ns]"`).
    fn name(&self) -> &str;

    /// Storage element size in bytes — an estimate for variable-size types.
    /// Feeds the same cost/chunk-budget math as `PrimitiveType::itemsize`.
    fn itemsize(&self) -> usize;

    /// The binary-op capability, or `None` if this dtype advertises no
    /// binary ops (then every `binary()` involving it is a plan-time error).
    /// FIXME: we may need to be smarter here, but for now... this is OK.
    fn binary(&self) -> Option<&dyn BinaryCapability>;

    /// Allocate a writable host array of `shape` with unspecified contents —
    /// dtype-driven construction (`np.empty(shape, dtype)`). Lives on the
    /// dtype because no array exists yet to dispatch through; sinks populate
    /// the result via [`ArrayData::write_range`].
    fn empty(&self, shape: &[usize]) -> Box<dyn ArrayData>;

    /// Allocate a writable host array of `shape` filled with `fill` —
    /// `np.full(shape, fill, dtype)`. Infallible like [`Self::empty`]:
    /// `fill` belonging to this dtype is a caller-validated precondition,
    /// so the satellite downcast is a genuine invariant (`expect`). Serves
    /// sink prefill (extension fills) and rank-0 literal materialization.
    fn full(&self, fill: &dyn ExtensionScalar, shape: &[usize]) -> Box<dyn ArrayData>;
}

empty/full live on the dtype (not the array) because they construct from nothing — Array API empty(shape, dtype) — and sinks need them before any array exists.

Binary Ops

Binary ops dispatch through a capability modeled directly on Python’s __add__/__radd__ operand-coercion protocol. Because planning precedes materialization, BinaryCapability::apply runs with no brick: it infers the output dtype, inserts operand casts, and returns a built ArrayExpr.

/// Build `this <op> other` when this dtype owns the op. The direct analogue
/// of Python's `__add__`/`__radd__` operand-coercion protocol;
/// `apply` runs with no brick — it infers the output dtype, inserts operand casts,
/// and returns a built [`ArrayExpr`].
///
/// Operands are [`Operand`]s, so scalars dispatch through the same protocol
/// (an extension-*scalar* lhs dispatches via its dtype).
///
/// Contract, mirroring Python exactly
/// (<https://docs.python.org/3/reference/datamodel.html#emulating-numeric-types>):
/// - `None`           ≙ `__op__` returned `NotImplemented` → try the other operand
/// - `Some(Ok(expr))` ≙ a handled `__op__`/`__rop__` — the full lowered subgraph
/// - `Some(Err(e))`   ≙ the op is mine, but the operands are invalid (a real
///   error, not a decline)
pub trait BinaryCapability {
    fn apply(
        &self,
        op: BinaryOp,
        this: &Operand,
        other: &Operand,
    ) -> Option<ArrayResult<ArrayExpr>>;
}

An extension crate is expected to handle both array and scalar operands.

Array payloads

A brick’s worth of extension data is the Extension arm of ArrayPayload:

/// Array-layer brick payload. The arms are the closed *kind* tag; the data
/// behind each arm answers the uniform [`ArrayData`] contract.
// Keep this in sync with [`ArrayPayloadKind`], [`WriteFrame`]
#[derive(Debug, Clone)]
pub enum ArrayPayload {
    Dense(DenseArray),
    /// Externally provided payload (datetime, categorical, string, geometry).
    Extension(Arc<dyn ArrayData>),
    // Sparse(Arc<dyn SparseArray>)
}

The crate hands core an Arc<dyn ArrayData> (the type its ExtensionDType::empty/full return). ArrayData is the structural contract every payload answers regardless of dtype:

/// The uniform contract over payload data — what every payload
/// kind supports regardless of storage representation.
/// TODO: break this up in to more super-traits when it makes sense.
///       this will let us be more explicit about the contract for each use-case
///       e.g. ArrayData is too much for SinkBuffer;
///       Note: traits cant have constructors, so `empty` is not here but on the DType.
///       DType is the only thing the LoweredSink holds.
pub trait ArrayData: Index + Send + Sync + Debug {
    /// Logical element dtype of this payload.
    fn dtype(&self) -> DType;

    /// Per-axis element extents.
    fn shape(&self) -> &[usize];

    /// Drop the given size-1 axes (ascending positions; extents pre-validated
    /// by the caller, [`ArrayPayload::squeeze`]). A buffer-sharing reshape.
    fn squeeze(&self, axes: &[usize]) -> BrixResult<ArrayPayload>;

    /// Write `data` into the element-space region `ranges` (one per axis) —
    /// the sink population path, only ever called on a buffer the sink
    /// exclusively owns (core-allocated, or via
    /// [`crate::extension::ExtensionDType::empty`]). Accept the payloads
    /// this data kind can ingest; reject the rest.
    fn write_range(&mut self, ranges: &[Range<usize>], data: &ArrayPayload) -> BrixResult<()>;

    /// The kernel-side downcast hook.
    fn as_any(&self) -> &dyn Any;
}

So an extension array implements, in order of how much it can lean on Arrow-style buffer sharing:

  • Structural indexing — Index - slice/oindex/vindex.
  • Shape manipulation — squeeze (drop size-1 axes).
  • Metadata — dtype and shape.
  • The sink path — write_range, called only on a buffer the sink exclusively owns; the impl accepts the payloads its kind can ingest and rejects the rest.
  • The kernel hook — as_any, so a satellite kernel can downcast back to its concrete array and compute on it.

Note

We may break this trait up in to a number of supertraits in the future.

Index

The Index supertrait:

/// Per-brick structural indexing. Every method returns an owned
/// [`ArrayPayload`]; `slice` is zero-copy by kind contract (`Dense` shares
/// the parent buffer through `ArcArray`, `Extension` impls buffer-share
/// Arrow-style), while `oindex`/`vindex` gather into fresh buffers.
pub trait Index {
    // TODO: we need an iindex - "integer index" too
    fn slice(&self, slicers: &[(usize, Slice)]) -> ArrayPayload;
    fn oindex(&self, indexers: &[(usize, Vec<usize>)]) -> ArrayPayload;
    fn vindex(&self, indexers: &[(usize, Vec<usize>)], produces: usize) -> ArrayPayload;
}

Core never matches on the concrete type: it indexes and writes through the trait, and hands the payload to satellite kernels — selected by the dtype’s capability — that downcast and do the actual work.

Scalars

The host-side rank-0 value mirrors the array payload:

/// A satellite-owned rank-0 value. Surfaces at the **host boundary** —
/// constructing a literal, or materializing a 0-d result (`.item()`) — and
/// as a binary operand ([`Operand::Scalar`]): the owning capability resolves
/// it at plan time, to its storage primitive or to an extension-dtype
/// literal node (see [`BinaryCapability`]).
pub trait ExtensionScalar: DynEqHash + Send + Sync + Debug {
    /// Recover the dtype from the value (e.g. `Category("Forest")` →
    /// its `CategoricalDType`).
    fn dtype(&self) -> Arc<dyn ExtensionDType>;
}

Example: a minutes dtype

This toy minutes dtype stores an i64 count per element. A full satellite spells out three structures — the dtype, the scalar, and the array payload — plus the binary capability that ties them together.

The dtype

The dtype is identity (name, itemsize) plus the capability gates and the empty/full constructors. No compute lives here — it dispatches through the capabilities and the payload:

/// Toy satellite dtype: an i64 count of minutes. `minutes ± minutes` and
/// `minutes * i64` are mine (built over the i64 storage, retagged);
/// everything else declines.
#[derive(Debug, PartialEq, Eq, Hash)]
struct Minutes;

impl Minutes {
    fn dtype() -> DType {
        DType::Extension(Arc::new(Minutes))
    }
}

impl ExtensionDType for Minutes {
    fn name(&self) -> &str {
        "minutes"
    }
    fn itemsize(&self) -> usize {
        8
    }
    fn binary(&self) -> Option<&dyn BinaryCapability> {
        Some(self)
    }
    fn empty(&self, shape: &[usize]) -> Box<dyn ArrayData> {
        Box::new(MinutesArray(ArrayD::zeros(IxDyn(shape))))
    }
    fn full(&self, fill: &dyn ExtensionScalar, shape: &[usize]) -> Box<dyn ArrayData> {
        let m = fill
            .as_any()
            .downcast_ref::<MinutesScalar>()
            .expect("caller validated: a minutes fill for the minutes dtype");
        Box::new(MinutesArray(ArrayD::from_elem(IxDyn(shape), m.0)))
    }
}

The scalar

The host-side rank-0 value is a literal count of minutes (5min); it only has to report its dtype so the binary protocol can route on it:

/// Toy satellite scalar: a literal count of minutes (`5min`).
#[derive(Debug, PartialEq, Eq, Hash)]
struct MinutesScalar(i64);

impl ExtensionScalar for MinutesScalar {
    fn dtype(&self) -> Arc<dyn ExtensionDType> {
        Arc::new(Minutes)
    }
}

The array payload

Its payload wraps an ArrayD<i64> and implements the ArrayData contract.

/// Toy satellite payload: minutes stored as `ArrayD<i64>`. A real impl
/// buffer-shares on `slice` (Arrow-style); this test impl copies.
#[derive(Debug)]
struct MinutesArray(ArrayD<i64>);

impl Index for MinutesArray {
    fn slice(&self, slicers: &[(usize, Slice)]) -> ArrayPayload {
        let v = slicers.iter().fold(self.0.view(), |mut v, (ax, s)| {
            v.slice_axis_inplace(
                Axis(*ax),
                ndarray::Slice::new(s.start as isize, Some(s.end as isize), s.step as isize),
            );
            v
        });
        ArrayPayload::Extension(Arc::new(MinutesArray(v.to_owned())))
    }

    fn oindex(&self, indexers: &[(usize, Vec<usize>)]) -> ArrayPayload {
        let out = indexers
            .iter()
            .fold(self.0.clone(), |out, (axis, positions)| out.select(Axis(*axis), positions));
        ArrayPayload::Extension(Arc::new(MinutesArray(out)))
    }

    fn vindex(&self, _indexers: &[(usize, Vec<usize>)], _produces: usize) -> ArrayPayload {
        unimplemented!("not exercised by these tests")
    }
}

impl ArrayData for MinutesArray {
    fn dtype(&self) -> DType {
        Minutes::dtype()
    }
    fn shape(&self) -> &[usize] {
        self.0.shape()
    }
    fn squeeze(&self, axes: &[usize]) -> BrixResult<ArrayPayload> {
        let v = axes.iter().rev().fold(self.0.view(), |v, &p| v.index_axis_move(Axis(p), 0));
        Ok(ArrayPayload::Extension(Arc::new(MinutesArray(v.to_owned()))))
    }
    fn write_range(&mut self, ranges: &[Range<usize>], data: &ArrayPayload) -> BrixResult<()> {
        let src = match data {
            ArrayPayload::Extension(e) => e
                .downcast_ref::<MinutesArray>()
                .ok_or(BrixError::DowncastError { expected: "MinutesArray" })?,
            _ => {
                panic!("minutes sink accepts minutes bricks only");
            }
        };
        let mut view = self.0.view_mut();
        for (axis, range) in ranges.iter().enumerate() {
            view.slice_axis_inplace(
                Axis(axis),
                ndarray::Slice::from(range.start as isize..range.end as isize),
            );
        }
        view.assign(&src.0);
        Ok(())
    }
    fn as_any(&self) -> &dyn Any {
        self
    }
}

as_any is the hook that lets a satellite kernel recover the concrete MinutesArray from the dyn ArrayData it was handed. The kernel selected for the minutes dtype calls payload.downcast_ref::<MinutesArray>() (which routes through as_any) to get back its i64 buffer.

Binary ops

The toy minutes dtype owns minutes ± minutes and minutes * i64, and declines everything else.

Since the underlying storage is ArrayD<i64> this code reuses the internal i64 path. Note the use of ArrayError::ExtensionDowncastError for nice error handling.

fn retag(e: &Arc<ArrayExpr>, dtype: DType) -> Arc<ArrayExpr> {
    Arc::new(ArrayExpr { dtype, ..e.as_ref().clone() })
}

impl BinaryCapability for Minutes {
    /// Supports minutes ± minutes and minutes * i64, and declines everything else.
    fn apply(
        &self,
        op: BinaryOp,
        this: &Operand,
        other: &Operand,
    ) -> Option<ArrayResult<ArrayExpr>> {
        use BinaryOp::*;

        // Can we handle the requested operation?
        let other_dtype = other.dtype();
        let handled = match op {
            // minutes ± minutes
            Add | Sub => other_dtype.as_extension().is_some_and(|e| e.is::<Minutes>()),
            // minutes * i64
            Mul => matches!(other_dtype, DType::Primitive(PrimitiveType::I64)),
        };
        if !handled {
            // don't know what to do here, bail.
            return None;
        }

        // Resolve operands to their i64 storage and reuse the core's binary op path.
        let as_i64 = |o: &Operand| -> ArrayResult<Operand> {
            Ok(match o {
                Operand::Expr(e) => Operand::Expr(retag(e, DType::I64)),
                Operand::Scalar(ScalarValue::Primitive(v)) => Operand::from(*v),
                Operand::Scalar(ScalarValue::Extension(s)) => {
                    // A scalar that advertises the minutes dtype but isn't a
                    // `MinutesScalar` is a malformed operand
                    let m = s
                        .downcast_ref::<MinutesScalar>()
                        .ok_or(ArrayError::ExtensionDowncastError { expected: "MinutesScalar" })?;
                    Operand::from(Value::I64(m.0))
                }
            })
        };
        Some((|| -> ArrayResult<ArrayExpr> {
            let expr = ArrayExpr::binary(op, as_i64(this)?, as_i64(other)?)?;
            Ok(ArrayExpr { dtype: Minutes::dtype(), ..expr })
        })())
    }
}

Executing end-to-end

With the dtype and payload in hand, a minutes array runs through the full pipeline like any built-in dtype.

#[test]
fn in_memory_source_extension_round_trip() {
    use brix::executor::SyncExecutor;
    use brix_array::{Array, ArrayLike};

    // Wrap an extension payload in a logical Array — positional, dim-named "t".
    let values = ArrayD::from_shape_fn(IxDyn(&[4]), |ix| ix[0] as i64);
    let arr = Array::from_payload(
        ArrayPayload::Extension(Arc::new(MinutesArray(values.clone()))),
        &["t"],
    );

    // `load` runs the whole optimize → sink → lower → plan → execute pipeline
    // and re-wraps the materialized payload as a fresh in-memory Array. The
    // budget (2 * 8 bytes) forces two bricks of 2 elements, so this exercises
    // brick-wise reassembly of extension data.
    let loaded = arr.load(SyncExecutor, 2 * 8).unwrap();

    // `payload` hands back the reassembled buffer; downcast to MinutesArray.
    let ArrayPayload::Extension(e) = loaded.payload().expect("load yields an in-memory array")
    else {
        panic!("expected an extension payload out of the pipeline");
    };
    let minutes = e.downcast_ref::<MinutesArray>().expect("payload is MinutesArray");
    assert_eq!(minutes.0, values);
}
